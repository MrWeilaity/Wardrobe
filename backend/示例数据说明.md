## 使用说明

### 自动加载

系统启动时会自动加载示例数据到MySQL数据库。数据文件位于 `backend/src/main/resources/data.sql`。

### 登录测试

可使用以下任一账户登录系统：

```
用户名: zhangsan
密码: password123
```

或

```
用户名: lisi
密码: password123
```

或

```
用户名: wangwu
密码: password123
```

### 数据特点

1. **真实性**: 所有数据模拟真实场景，包含实际品牌、合理价格、真实搭配
2. **多样性**: 涵盖不同季节、场合、风格的衣物和穿搭
3. **关联性**: 穿搭方案和旅行计划与衣物数据完整关联
4. **中文化**: 所有描述均使用中文，符合国内用户习惯

### 清空示例数据

如需清空示例数据，可登录MySQL执行：

```sql
-- 清空所有表（按依赖顺序）
DELETE FROM travel_clothing;
DELETE FROM outfit_clothing;
DELETE FROM travel_plans;
DELETE FROM outfits;
DELETE FROM clothing_tags;
DELETE FROM clothing;
DELETE FROM user_roles;
DELETE FROM users;

-- 重置自增ID
ALTER TABLE users AUTO_INCREMENT = 1;
ALTER TABLE clothing AUTO_INCREMENT = 1;
ALTER TABLE outfits AUTO_INCREMENT = 1;
ALTER TABLE travel_plans AUTO_INCREMENT = 1;
```

### 禁用自动加载

如不需要示例数据，可在 `application.yml` 中注释掉以下配置：

```yaml
spring:
  sql:
    init:
      mode: always  # 改为 never
      # data-locations: classpath:data.sql  # 注释掉此行
```

## 常见问题

### Q1: 为什么看不到示例数据？
A: 检查以下几点：
1. 确认MySQL数据库已创建
2. 确认`application.yml`中配置了数据加载
3. 查看后端启动日志，确认SQL执行成功
4. 使用正确的用户名和密码登录

### Q2: 如何添加更多示例数据？
A: 编辑`data.sql`文件，按照现有格式添加INSERT语句，注意ID的连续性和外键关联。

### Q3: 示例数据会影响生产环境吗？
A: 建议仅在开发和演示环境使用示例数据。生产环境应禁用自动数据加载，使用真实用户数据。

### Q4: 密码是真的BCrypt加密吗？
A: 示例中的密码哈希仅用于演示。实际使用时，请通过注册接口创建用户，系统会自动使用真正的BCrypt加密。
